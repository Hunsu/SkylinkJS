Constructor

Socket ( config ,   listener ,   ) Version 0.6.0

Defined in: source/Socket.js:1

This Socket constructor cannot be instantiable, meaning you cannot create a new Socket class object that would require certain parent dependencies functionalities. E.g. DataChannel class relies on Peer class.

To create a Socket class object, these are the required parameters to be provided.

Parameters:
  • config : type JSON

    The socket connection configuration.

    • server : type String

      The socket signaling server.

    • httpsPortList : type Array

      The list of HTTPS ports that the socket connection would use. It uses the first port and fallbacks to the next alternative port when connection fails.

    • httpPortList : type Array

      The list of HTTP ports that the socket connection would use. It uses the first port and fallbacks to the next alternative port when connection fails.

    • type : type String

      The type of socket connection. There are two types:

      • "WebSocket" indicates a WebSocket connection.
      • "XHRPolling" indicates a LongPolling connection.
  • listener : type Function

    The listener function.
