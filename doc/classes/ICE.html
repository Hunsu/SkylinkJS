parseIceConnectionState()

Parses the received ICE connection state and updates to a new version to handle the differences received from cross-browsers. State should go from checking > connected > completed.
parseICEServers()

Handles the ICE servers received based on the options set by user and parses the differences for iceServer format for cross-browsers.
Format of an ICE server:

  • STUN is structured like { url: 'stun:hosturl' }.
  • TURN is structured like { url: 'turn:username@hosturl', credential: 'xxx' } and { url: 'turn:hosturl', username: 'username', credential: 'xxx' } for Firefox browsers.
popCandidate()

Adds all ICE candidates that is received before setRemoteDescription is called. It retrieves candidates from the queueCandidate property array in the peer connection object.
queueCandidate()

Queues ICE candidates that is received before setRemoteDescription is called. It stores in the queueCandidate property array in the peer connection object.

The revised versions of ICE connection states to handle the differences cross-browsers of different states. This was to feedback to various users the completion state of the ICE connection.
